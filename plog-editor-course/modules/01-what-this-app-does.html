Real code: where a new element is born

CODE 

function createElement(type, patch = {}) {
  const layout = canvasLayout();
  const prev = getInsertionAnchor() || state.elements[state.elements.length - 1];
  const spacingBefore = patch.spacingBefore ?? defaultSpacingBefore(type, prev?.type);
  const flow = getFlowPlacement(type, spacingBefore);

  const base = {
    id: uid(type),
    type,
    x: flow.x,
    y: flow.y,
    width: layout.contentWidth,
    height: 120,
    content: "",
    html: "",
    placeholder: "Write your story...",
    spacingBefore,
    style: {
      fontSize: 60,
      color: "#1f1f22",
      radius: 10,
      fontFamily: "fangzheng",
      fontWeight: 300,
      rotation: 0,
      brightness: 100,
      contrast: 100,
      grayscale: 0,
      frame: "none",
    },
    src: "",
    aspectRatio: null,
  };
PLAIN ENGLISH

This function is the template factory. Every new thing you add starts here.

It looks at the current canvas layout and at the thing that came before, so new blocks can land in a sensible place.

Then it creates one base object with an ID, position, size, empty content, and default visual settings.

Nothing is visible yet. This is still just data waiting to be rendered.

💡
Why this matters for steering AI

If you know there is one factory function for new elements, you can tell an AI agent “change the defaults in createElement” instead of vaguely saying “make new blocks look different.”
