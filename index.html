II. Why it feels different 002 / 004

Surface

Landing calm, editor focus.

The first page explains the product and sets tone. The editor page keeps the workflow dense, but the chrome becomes more premium, card-based, and modern.

Theme logic

Light gets the editorial warmth. Dark gets the same hierarchy.

The warm-paper language maps naturally to light mode. The dark theme keeps the same spacing, typography, and control hierarchy without becoming a literal inverted poster.

Interaction

Buttons and drawers read like product UI, not raw controls.

The goal is not maximal decoration. It is higher-quality affordance: better grouping, clearer primary actions, more deliberate cards, and less “utility panel” feel.

III. Surface language 003 / 004

Editor shell

Sticky topbar, stronger panels, calmer cards.

  • Document drawer reads like a library, not a raw select box.
  • Canvas stage feels framed and intentional.
  • Inspector groups become readable product sections.

Theme bridge

One design language, two moods.

  • Light mode borrows editorial warmth from Open Design.
  • Dark mode keeps the same proportions and emphasis structure.
  • Accent color remains controlled instead of flooding the page.
